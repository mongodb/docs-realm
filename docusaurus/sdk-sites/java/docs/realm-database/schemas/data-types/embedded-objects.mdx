**Embedded objects** are objects nested inside of parent objects. An
embedded object inherits the lifecycle of its parent object and cannot
exist without the parent. Realm automatically deletes
embedded objects if:

- you delete the parent object
- you overwrite the embedded object with a new instance of that object

Embedded objects are similar to relationships, but they provide
additional constraints and map more naturally to the denormalized
MongoDB document model.

## Embedded Object Data Models

You can use the same embedded object type in multiple parent
object types and you can embed objects inside of other embedded objects.
Embedded objects cannot have a primary key.

:::note Cascading Deletes for Embedded Objects
When you delete a Realm object, Realm automatically deletes any
embedded objects referenced by that object. Any objects that your
application must persist after the deletion of their parent object
should use relationships instead.
:::

To define an embedded object, derive a class from `RealmObject`
and set the `embedded` property of the
[`RealmClass`](https://docs.mongodb.com/realm/sdk/java/api/io/realm/annotations/RealmClass/)
annotation to `true`. You can reference an embedded object type from parent
object types in the same way as you would define a relationship:
