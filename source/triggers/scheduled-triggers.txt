Scheduled Triggers

   :local:

Overview
--------

Scheduled {+triggers-short+} allow you to execute server-side logic on a
:doc:`regular schedule that you define </triggers/cron-expressions>`.
You can use scheduled {+triggers-short+} to do work that happens on a periodic
basis, such as updating a document every minute, generating a nightly
report, or sending an automated weekly email newsletter.

Create a Scheduled Trigger
--------------------------

.. tabs-realm-admin-interfaces::

   tabs:
     - id: ui
       content: |
         To create a scheduled {+trigger-short+} in the {+ui+}:

         1. Click :guilabel:`Triggers` under :guilabel:`MongoDB
            Cluster` in the left-hand navigation.

         2. Select :guilabel:`Scheduled Triggers` from the {+trigger-short+} type
            dropdown menu.

         3. Click :guilabel:`Add a Trigger` to open the {+trigger-short+}
            configuration page.

         4. Enter :ref:`configuration values
            <scheduled-trigger-configuration>` for the {+trigger-short+}.
            You can configure two types of {+trigger-short+} schedules:
            :guilabel:`Basic` or :guilabel:`Advanced`.

            - A :guilabel:`Basic` schedule executes the {+trigger-short+}
              periodically based on a single unit of time and an
              interval, such as "every five minutes" or "every Monday".

            - An :guilabel:`Advanced` schedule executes the {+trigger-short+}
              based on a custom :doc:`CRON expression
              </triggers/cron-expressions>` that you define.

         5. Click :guilabel:`Save`.

         .. cssclass:: bordered-figure
         .. figure:: /images/trigger-example-scheduled.png

     - id: cli
       content: |
         To create a scheduled {+trigger-short+} with :doc:`realm-cli
         </deploy/realm-cli-reference>`:

         1. Add a scheduled {+trigger-short+} :ref:`configuration file
            <scheduled-trigger-configuration>` to the ``triggers``
            subdirectory of a local application directory.

            .. note::

               You cannot create a {+trigger-short+} that runs on a
               :guilabel:`Basic` schedule using Realm CLI. All imported
               scheduled {+trigger-short+} configurations must specify a
               :doc:`CRON expression </triggers/cron-expressions>`.

         2. :doc:`Import </deploy/deploy-cli>` the
            application directory into your application.

            .. code-block:: shell

               realm-cli import

Scheduled Trigger Configuration
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

.. tabs-realm-admin-interfaces::

   tabs:
     - id: cli
       content: |
         Scheduled {+trigger-short+} configuration files have the following form:

         .. cssclass:: wide-container
         .. code-block:: none
            :caption: ``/triggers/<triggers name>.json``

            {
               "type": "SCHEDULED",
               "name": "<Trigger Name>",
               "function_name": "<Trigger Function Name>",
               "config": {
                 "schedule": "<CRON expression>"
               },
               "disabled": <boolean>
            }

     - id: ui
       content: |
         Scheduled {+triggers-short+} have the following configuration parameters:

.. cssclass:: auth-table
.. list-table::
   :header-rows: 1
   :widths: 15 30

   * - Field
     - Description

   * - :guilabel:`Trigger Type`

       .. tab-content:: tabs-realm-admin-interfaces
          :tab-id: cli

          | ``type: <string>``

     - Required. The type of the {+trigger-short+}.

       .. tab-content:: tabs-realm-admin-interfaces
          :tab-id: cli

          For scheduled {+triggers-short+}, this value should be set to
          ``SCHEDULED``.

   * - :guilabel:`Trigger Name`

       .. tab-content:: tabs-realm-admin-interfaces
          :tab-id: cli

          | ``name: <string>``

     - Required. The name of the {+trigger-short+}.

   * - :guilabel:`Linked Function`

       .. tab-content:: tabs-realm-admin-interfaces
          :tab-id: cli

          | ``function_name: <string>``

     - Required. The name of the {+function+} that the {+trigger-short+}
       executes whenever it fires.

       .. note::

          A scheduled {+trigger-short+} does not pass any arguments to its linked
          {+function-short+}.

   * - :guilabel:`Schedule`

       .. tab-content:: tabs-realm-admin-interfaces
          :tab-id: cli

          | ``config.schedule: <string>``

     - Required. The :doc:`CRON expression </triggers/cron-expressions>`
       that {+backend+} uses to determine when to fire the {+trigger-short+}.
